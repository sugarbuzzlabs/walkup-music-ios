  <action>
    Create Codable, Identifiable structs:

    Player:
    - id: UUID
    - name: String
    - jerseyNumber: Int
    - songFileName: String? (filename of imported MP3 in app sandbox, nil if no song assigned)
    - songStartTime: TimeInterval (default 0.0 — seconds offset to start playback, e.g. skip to chorus)
    - createdAt: Date

    Lineup:
    - id: UUID
    - name: String (e.g. "Game 1", "vs Tigers")
    - playerIDs: [UUID] (ordered array — position in array = batting order)
    - createdAt: Date

    AppSettings:
    - autoStopDuration: TimeInterval (default 10.0 seconds)
    - defaultVolume: Float (default 0.8, range 0.0-1.0)

    All structs should conform to Codable, Identifiable, and Hashable. Use sensible defaults via init() with default parameter values.
  </action>
  <verify>Project builds with no errors after adding model files</verify>
  <done>Three model files exist, all compile, all conform to Codable + Identifiable + Hashable</done>

<task type="auto">
  <name>Task 2: Build JSON persistence layer</name>
  <files>
    WalkoutDJ/Services/StorageManager.swift
  </files>
  <action>
    Create a StorageManager class (ObservableObject) that:

    1. Stores data as JSON files in the app's Documents directory:
       - players.json → [Player]
       - lineups.json → [Lineup]
       - settings.json → AppSettings
    2. @Published properties:
       - players: [Player]
       - lineups: [Lineup]
       - settings: AppSettings
    3. Methods:
       - savePlayers() / loadPlayers()
       - saveLineups() / loadLineups()
       - saveSettings() / loadSettings()
       - addPlayer(_ player: Player)
       - updatePlayer(_ player: Player)
       - deletePlayer(id: UUID) — also removes from all lineups and deletes song file if present
       - addLineup(_ lineup: Lineup)
       - updateLineup(_ lineup: Lineup)
       - deleteLineup(id: UUID)
       - importSongFile(from sourceURL: URL) -> String? — copies file to app's songs directory, returns filename
       - deleteSongFile(named fileName: String)
       - songFileURL(for fileName: String) -> URL
    4. Create a "Songs" subdirectory in Documents for imported MP3 files
    5. Load all data on init()
    6. Auto-save after mutations (call save after add/update/delete methods)

    Use @MainActor for the class since it publishes UI state. Handle file errors gracefully with print() logging (no crashes). If files don't exist on first launch, start with empty arrays and default settings.

    Inject via .environment() or .environmentObject() — add it to WalkoutDJApp.swift as @StateObject and pass via .environmentObject().
  </action>
  <verify>Project builds; manually verify StorageManager initializes without crash by building and running</verify>
  <done>StorageManager compiles, initializes in app, creates Documents/Songs directory on first launch, can encode/decode all models to JSON</done>
</task>
